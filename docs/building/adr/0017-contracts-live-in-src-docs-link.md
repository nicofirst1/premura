# Contracts live in `src` beside their code; docs link, never duplicate

> **Status:** Accepted — 2026-07-15

Premura's architecture contracts had accumulated in `docs/building/architecture/`, several of them duplicating or drifting from the code they govern (the `DISCLOSURE_RUBRIC.md` docs copy had already diverged from its bundled skill copy despite a byte-identity drift-guard test). The principle, applied repo-wide: **every contract has exactly one physical home, next to the code it governs under `src/premura/`, and every other file links to that home rather than restating it.** Five decisions follow. (1) **Each contract co-locates with its package**, extending the existing `src/premura/engine/CONTRACT.md` and `src/premura/parsers/CONTRACT.md` pattern: `OPERATING_ROLES.md` and `HUMAN_FACING.md` → `src/premura/ui/` (Stage 4 owns the role registry and interview tracks); `INTAKE_DIMENSIONS.md` → `src/premura/engine/` (the add-a-resolver rule over `engine/views/`); `UPDATE_STRATEGY.md` and `PROFILE_AND_INTAKE_CONTRACT.md` (plus its four machine-readable `*.yaml` companions, now under `src/premura/store/profile_intake_contracts/`) → `src/premura/store/` (the storage adapter, migrations, and load path live there); `AI_CHAT_RECALL_CONTRACT.md` and `PARSER_CONTRIBUTING.md` → `src/premura/parsers/`; `AUDIT_CONSUMER_CONTRACT.md` → `src/premura/` beside `trace.py`, which produces the Session Disclosure object it specifies; `DISCLOSURE_RUBRIC.md` → its skill dir `src/premura/skills/human-facing-teaching/`, where it already shipped as package data. (2) **No duplication and no drift guards.** The `DISCLOSURE_RUBRIC.md` docs copy and its byte-identity drift-guard test are deleted; the bundled skill copy is the sole home, matching the `AUDIT_RUBRIC.md` precedent (skill-dir-only, linked to from docs). A guard that exists only to keep two copies in sync is a tax that a single home removes. (3) **Symlinks were rejected.** Package data must live under the package root so `install_skills()` and `importlib.resources` can reach it; a symlink pointing outside the package (or its target) is excluded from `hatchling` wheels and breaks on Windows checkouts, so it would work only in a dev clone. (4) **`docs/building/architecture/` collapses to `STAGES.md`** as the single cross-cutting map and index, which points at every co-located contract. STAGES stays in `docs/` because it spans every package and belongs to none. (5) **Docs contain links, not content.** A doc may summarize a contract in one line but must link to its `src` home; on any disagreement the `src` file wins. This supersedes the older "the docs copy is authoritative" framing wherever it appeared. Concept lineage: this is the repo-wide application of the single-home rule in [`DOCTRINE.md`](../../shared/DOCTRINE.md) ("does this doc restate a fact that already has a home?").
