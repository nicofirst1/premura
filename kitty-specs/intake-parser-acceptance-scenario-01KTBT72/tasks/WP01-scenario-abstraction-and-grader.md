# WP01 — Scenario abstraction + drawer-parametric grader

## Objective

Introduce the **`Scenario`** abstraction and an injected **`DrawerGradingStrategy`**, then
refactor `grade()` so its three rules (`loaded`, `runtime_valid`, `honest_about_gaps`) are
computed via the scenario's strategy with **no per-drawer branch** in the shared body —
while keeping the observation verdict **byte-for-byte identical** (C-004). This is the
foundation every other WP builds on. It ships intake-ready seams but **no intake code**;
the only registered scenario here is the observation one.

## Context

- Read [spec.md](../spec.md) (FR-001, FR-010, C-004, NFR-005) and
  [research.md](../research.md) D1/D2, plus
  [contracts/scenario-contract.md](../contracts/scenario-contract.md) and
  [contracts/drawer-grading-contract.md](../contracts/drawer-grading-contract.md).
- Today's grader (`src/premura/harness/grader.py`): `grade(provenance, warehouse_conn,
  fixture_manifest)` computes three rules. `_grade_loaded` reads the `_BOUNDARY` **fact**
  tables; `_grade_runtime_valid` delegates to `check_runtime_contract`; `_grade_honest_about_gaps`
  reconciles manifest fields vs (loaded ∪ declared). The verdict schema is
  `{passed, rules:{loaded, runtime_valid:{passed,violations}, honest_about_gaps}}`,
  arrays sorted, **no ids, no timestamps**.
- The hard rule: the generalized `grade()` body must contain **no** `if observation/elif
  intake`, no drawer/scenario/table name. All divergence lives behind the strategy.

## Subtasks

### T001 — Capture the golden observation verdict (do this FIRST)

**Purpose**: Freeze current observation behavior before touching `grade()` so the refactor
is provably non-regressing.

**Steps**:
1. Using the committed synthetic observation fixture + manifest the existing seam uses
   (`tests/fixtures/session_log/fitbit_heart_rate_synthetic.csv` + `fixture_fields.yaml`),
   run the *current* grading path and capture the exact verdict dict.
2. Store it as a committed golden in `tests/test_observation_scenario_golden.py` (an inline
   literal or a small JSON loaded by the test). It must be the pre-refactor output.

**Validation**: the golden is captured from `master`'s current grader, not hand-written.

### T002 — Define `Scenario` + `DrawerGradingStrategy` + registry primitives

**Purpose**: The bounded abstraction (FR-001).

**Steps** (in new `src/premura/harness/scenario.py`):
1. `@dataclass(frozen=True) class Scenario` with: `name: str`, `source_path: Path`,
   `manifest_path: Path`, `reference_parser: str` (import target), `strategy: DrawerGradingStrategy`.
2. Define `DrawerGradingStrategy` as a `Protocol` with three methods:
   - `boundary_truth(conn) -> BoundaryTruth` (loaded row count + present keys),
   - `runtime_check(provenance, conn) -> ContractCheckResult`,
   - `gap_set(manifest, provenance, boundary_truth) -> list[str]` (silent-drop fields).
3. Provide light registry primitives: a `register`/`builtin` mechanism is **not** required
   here (the registry that lists ≥2 scenarios is WP04's `scenario_registry.py`). Expose an
   `observation_scenario()` factory so callers (and WP04) can obtain the observation scenario.

> Keep `scenario.py` import-light (no Ollama, no network). It is imported by the grader.

### T003 — Implement the observation `DrawerGradingStrategy`

**Purpose**: Wrap today's logic verbatim so the observation verdict is preserved (C-004).

**Steps**:
1. Move (do not rewrite) the current `_grade_loaded` fact-table logic, the
   `check_runtime_contract` call, and the honesty reconcile into an `ObservationStrategy`
   exposing the three protocol methods.
2. `observation_scenario()` returns a `Scenario` wired to this strategy + the committed
   observation fixture/manifest.

### T004 — Refactor `grade()` to compute rules via the strategy

**Purpose**: One generic grader, drawer-injected (NFR-005).

**Steps**:
1. Change `grade()` to `grade(provenance, warehouse_conn, fixture_manifest, *, strategy=<observation default>)`.
   The **default keeps existing call sites working** (the seam test still passes a manifest
   and gets observation behavior) — important so WP01 is green without touching `live_trial.py`.
2. Body: call `strategy.boundary_truth(...)`, `strategy.runtime_check(...)`,
   `strategy.gap_set(...)`, then assemble the **unchanged** verdict schema. No drawer/table
   name appears in the body.
3. `contract_pass` persisted by callers stays the grader's recomputed `runtime_valid`.

### T005 — Golden-verdict regression test

**Purpose**: Prove C-004 / SC-006.

**Steps** (in `tests/test_observation_scenario_golden.py`):
1. Run the refactored `grade()` via the observation scenario over the committed fixture.
2. Assert the verdict equals the T001 golden **byte-for-byte** (same dict, sorted arrays).
