# WP03 — Alien source + manifest + reference intake parser

## Objective

Author the **synthetic, deliberately-alien** meals+supplements source, its **grader-only**
ground-truth manifest, and a **known-good reference intake parser** (the layer-1 operator).
Everything here is obviously-fake test data — **no PHI** (NFR-004), and the manifest must
never sit on any operator-visible path (C-005).

## Context

- Read [contracts/alien-source-and-manifest-contract.md](../contracts/alien-source-and-manifest-contract.md)
  and [research.md](../research.md) D5.
- Intake load path to target: `IntakeBatch` (`src/premura/parsers/base.py:282`) →
  `persist_intake_batch` (`src/premura/store/profile_intake.py:296`). Events:
  `NutritionIntakeInput` / `SupplementIntakeInput` (each needs a `dedupe_key`,
  `source_id`, and a matching `source_descriptor`).
- Mirror the existing intake fixtures' *spirit* but **not** their shape
  (`tests/fixtures/intake/nutrition_log.json` / `supplement_log.json` are JSON; this source
  is a different, foreign CSV).

## Subtasks

### T009 — `alien_intake.csv`

**Purpose**: A source structurally unlike all built-in sources and unlike the JSON intake
fixtures (FR-002).

**Steps** — author `tests/fixtures/intake_scenario/alien_intake.csv` with:
1. One CSV, both meals and supplements, distinguished by a `kind` column (`meal` / `supp`).
2. Foreign column names: `logged_at_us`, `kind`, `item`, `qty`, `qty_uom`, `note`.
3. `logged_at_us` = **epoch microseconds** (not ISO). Include a row whose local-day differs
   from its UTC date (crosses local midnight).
4. Mixed/non-SI units in `qty_uom`: e.g. `oz`, `Cal`, `IU`, `mcg`.
5. Coverage: ≥1 `meal` row, ≥1 `supp` row, the midnight-crossing row, and the `note` column
   carrying free text that has **no canonical home** (the intended declared-gap).
6. All values invented and obviously fake.

### T010 — `alien_intake_manifest.yaml` (grader-only)

**Purpose**: The ground-truth map the grader reconciles against (never operator-visible, C-005).

**Steps** — author `tests/fixtures/intake_scenario/alien_intake_manifest.yaml`:
1. Per source column: `source_column`, `drawer` (`intake`/`observation`),
   `canonical_home` (or `null` for the gap), `expected_outcome` (`loaded`/`declared_gap`).
2. `note` → `expected_outcome: declared_gap` (the SC-004 case).
3. Add a short header comment: **"GRADER-ONLY — never expose to an operator (C-005)."**

### T011 — `reference_intake_parser.py`

**Purpose**: The layer-1 known-good operator (FR-004).

**Steps** — author `tests/fixtures/intake_scenario/reference_intake_parser.py`:
1. A parser class exposing `parse(self, path) -> ParseOutput` (or `IntakeBatch`) that:
   - reads the CSV, routes `kind=meal` → `NutritionIntakeInput`, `kind=supp` →
     `SupplementIntakeInput`;
   - decodes `logged_at_us` (epoch µs) to the event timestamp (local-day aware);
   - converts `qty`/`qty_uom` units to the intake quantity representation;
   - registers a `source_descriptor` for every event `source_id`;
   - routes everything to the **intake** drawer (never `hp.fact_*`);
   - declares the `note` column as a gap via `unmapped_metrics` / `skipped_rows` —
     **never silently dropped**.
2. Expose module-level `MAPPED_SOURCE_COLUMNS` = the columns the parser consumed (used by
   the self-reconcile gate; the gate never infers it from the batch — C-005).
3. Returned batch must `validate()` and persist without raising.
4. Add an `__init__.py` if the fixtures dir needs it for import.
