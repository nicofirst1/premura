# WP02 тАФ Intake runtime_valid checker

## Objective

Implement `check_intake_runtime_contract` тАФ the **bounded** runtime-valid clause set for an
intake parser, defined against the **real** `IntakeBatch` surface. This is the intake
analogue of the shipped observation `check_runtime_contract`, and explicitly **not** the
full parser-review contract, and **not** a fake canonical-metric mirror (FR-010).

## Context

- Read [contracts/intake-runtime-contract.md](../contracts/intake-runtime-contract.md) and
  [research.md](../research.md) D3.
- Ground truth in code: `IntakeBatch` (`src/premura/parsers/base.py:282-330`) has
  `nutrition_events`, `supplement_events`, `source_descriptors`, per-event `source_id` +
  `dedupe_key`, `unmapped_metrics`, `skipped_rows`, and `validate()`. It has **no**
  `declared_metrics` / `emitted_metric_ids` / `dim_metric` (those are `IngestBatch` only).
- The observation checker to mirror in *shape* (not clauses): `check_runtime_contract`
  (`src/premura/parsers/contract_check.py`) returns `ContractCheckResult(runtime_valid: bool,
  violations: list[str])`, violations as sorted `"<clause>: <detail>"` strings.

## Subtasks

### T006 тАФ Implement `check_intake_runtime_contract`

**Purpose**: The three bounded intake clauses.

**Steps** (new `src/premura/harness/intake_contract_check.py`):
1. `clause parser_imports_and_parses` тАФ given the operator's parser is already run and a
   batch produced, this clause records a violation if the produced object is not an
   `IntakeBatch` (or `ParseOutput` carrying one). (Import/parse *raising* is surfaced by the
   harness as a parser error; this clause guards the "produced the wrong shape" case.)
2. `clause batch_validates` тАФ call `IntakeBatch.validate()`; on `ValueError` record a
   violation `batch_validates: <detail>`. This enforces the intake declared/emitted coherence
   on the **source dimension** (every event `source_id` is covered by a `source_descriptor`)
   plus `dedupe_key` uniqueness.
3. `clause persisted_without_raising` тАФ accept a captured boolean / error from the harness's
   persist step (the checker does not itself run persistence; it grades the captured outcome,
   mirroring how the observation clause grades `ingest_run` success). Record
   `persisted_without_raising: <detail>` on failure.

> Do **not** add a `declared_equals_emitted` metric clause or a `dim_metric` clause тАФ those
> have no intake counterpart (FR-010). The contract test asserts their absence.

### T007 тАФ Return a `ContractCheckResult`-shaped result

**Steps**:
1. Reuse the existing `ContractCheckResult` dataclass (import it) so the verdict schema's
   `runtime_valid.violations` shape is identical to the observation path.
2. `runtime_valid = (not violations)`; `violations` sorted.

### T008 тАФ Tests

**Steps** (`tests/test_intake_runtime_contract.py`):
1. A clean intake batch тЖТ `runtime_valid=True`, empty violations.
2. Missing `source_descriptor` for an event `source_id` тЖТ `batch_validates` violation.
3. Duplicate `dedupe_key` тЖТ `batch_validates` violation.
4. A simulated persist failure тЖТ `persisted_without_raising` violation.
5. **Clause-set assertion (FR-010 / SC-008):** the set of clause names equals exactly the
   three intake clauses; assert there is **no** `declared_equals_emitted` /
   `declared_exist_in_dim_metric` / `no_derived_emitted` clause (guards against drifting
   toward the observation or full-review contract).
