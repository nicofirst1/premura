# WP02 — Intake runtime_valid checker

## Objective

Implement `check_intake_runtime_contract` — the **bounded** runtime-valid clause set for an
intake parser, defined against the **real** `IntakeBatch` surface. This is the intake
analogue of the shipped observation `check_runtime_contract`, and explicitly **not** the
full parser-review contract, and **not** a fake canonical-metric mirror (FR-010).

## Context

- Read [contracts/intake-runtime-contract.md](../contracts/intake-runtime-contract.md) and
  [research.md](../research.md) D3.
- Ground truth in code: `IntakeBatch` (`src/premura/parsers/base.py:282-330`) has
  `nutrition_events`, `supplement_events`, `source_descriptors`, per-event `source_id` +
  `dedupe_key`, `unmapped_metrics`, `skipped_rows`, and `validate()`. It has **no**
  `declared_metrics` / `emitted_metric_ids` / `dim_metric` (those are `IngestBatch` only).
- The observation checker to mirror in *shape* (not clauses): `check_runtime_contract`
  (`src/premura/parsers/contract_check.py`) returns `ContractCheckResult(runtime_valid: bool,
  violations: list[str])`, violations as sorted `"<clause>: <detail>"` strings.

## Subtasks

> **Seam pinned (review fix).** The checker's *evidence* and its *producer* both live in
> this WP, so there is no cross-WP gap. The checker consumes a captured **runner outcome**:
> the envelope's existing `status` (`"ok"`/`"error"`) plus a **stage-tagged `error`** string
> (`"parse: …"` / `"validate: …"` / `"persist: …"`). No new envelope field, no change to the
> frozen session-log-substrate envelope schema — `status`/`error` are already allowed keys.
> Clause→evidence mapping:
> - `parser_imports_and_parses` ← `status=="ok"` OR `error` stage ≠ `parse` (a `parse:` error fails it)
> - `batch_validates` ← `error` stage ≠ `validate` (a `validate:` error fails it)
> - `persisted_without_raising` ← `error` stage ≠ `persist` (a `persist:` error fails it)
> WP06 (provenance) and WP07 (probe) merely *carry* `status`/`error` through for the live and
> failure paths; they do not define this seam.

### T006 — Implement `check_intake_runtime_contract`

**Purpose**: The three bounded intake clauses, computed from the captured runner outcome
(`status` + stage-tagged `error`) above.

**Steps** (new `src/premura/harness/intake_contract_check.py`):
1. `clause parser_imports_and_parses` — given the operator's parser is already run and a
   batch produced, this clause records a violation if the produced object is not an
   `IntakeBatch` (or `ParseOutput` carrying one). (Import/parse *raising* is surfaced by the
   harness as a parser error; this clause guards the "produced the wrong shape" case.)
2. `clause batch_validates` — call `IntakeBatch.validate()`; on `ValueError` record a
   violation `batch_validates: <detail>`. This enforces the intake declared/emitted coherence
   on the **source dimension** (every event `source_id` is covered by a `source_descriptor`)
   plus `dedupe_key` uniqueness.
3. `clause persisted_without_raising` — accept a captured boolean / error from the harness's
   persist step (the checker does not itself run persistence; it grades the captured outcome,
   mirroring how the observation clause grades `ingest_run` success). Record
   `persisted_without_raising: <detail>` on failure.

> Do **not** add a `declared_equals_emitted` metric clause or a `dim_metric` clause — those
> have no intake counterpart (FR-010). The contract test asserts their absence.

### T007 — Return a `ContractCheckResult`-shaped result

**Steps**:
1. Reuse the existing `ContractCheckResult` dataclass (import it) so the verdict schema's
   `runtime_valid.violations` shape is identical to the observation path.
2. `runtime_valid = (not violations)`; `violations` sorted.

### T008 — Tests

**Steps** (`tests/test_intake_runtime_contract.py`):
1. A clean intake batch → `runtime_valid=True`, empty violations.
2. Missing `source_descriptor` for an event `source_id` → `batch_validates` violation.
3. Duplicate `dedupe_key` → `batch_validates` violation.
4. A simulated persist failure → `persisted_without_raising` violation.
5. **Clause-set assertion (FR-010 / SC-008):** the set of clause names equals exactly the
   three intake clauses; assert there is **no** `declared_equals_emitted` /
   `declared_exist_in_dim_metric` / `no_derived_emitted` clause (guards against drifting
   toward the observation or full-review contract).

### T030 — Runner witnesses intake parse/validate/persist (the evidence producer)

**Purpose**: Make the in-sandbox runner *witness* the three intake outcomes so the checker
has real evidence — today the runner persists intake **without calling `validate()`**, so
`batch_validates` is unwitnessed (`src/premura/harness/ingest_runner.py:90-97`).

**Steps** (`src/premura/harness/ingest_runner.py`, additive — observation path unchanged):
1. When the parser yields intake, wrap the in-sandbox steps so each stage is witnessed and a
   failure sets `status="error"` with a **stage-tagged** `error`:
   - parse/normalize failure → `error="parse: <detail>"`;
   - call `intake.validate()` explicitly; on `ValueError` → `error="validate: <detail>"`;
   - `persist_intake_batch` failure → `error="persist: <detail>"`.
2. On success keep `status="ok"` and the existing intake fields (`unmapped_metrics`,
   `skipped_rows`). **No new top-level envelope key** — reuse `status`/`error` so the frozen
   `ingest-outcome-envelope.schema.json` (additionalProperties:false) still conforms and
   `tests/test_sandbox.py` stays green.
3. The runner is harness code witnessing the operator's batch — this is *not* trusting a
   parser self-report (the parser never sets these; the harness does).

**Validation**: a parse-stage, validate-stage, and persist-stage failure each produce the
correct stage-tagged `error`; a clean intake run yields `status="ok"`.
