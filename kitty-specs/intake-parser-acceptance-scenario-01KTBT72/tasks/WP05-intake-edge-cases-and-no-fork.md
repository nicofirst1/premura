# WP05 — Intake drawer + edge-case end-to-end suite

## Objective

Prove the **four deterministic** spec-named intake edge cases this WP owns — mis-filed row,
unmappable field, renamed-but-consumed field, intake-only vs both — each with a **real
end-to-end** harness run (drop → parse → load → grade), not a component-level assertion (D7),
plus the structural no-fork / ≥2-scenarios property.

> **Ownership boundary (review fix).** This WP does **not** own the fifth spec-named edge
> case, "live model produces a malformed parser." Its *deterministic* failure path is owned by
> **WP06** (`test_failure_path_record.py`, stub operator) and its *live* form by **WP07**
> (`test_live_trial_intake.py`). Do not duplicate it here.

## Context

- Depends on WP04 (`intake_strategy`, `intake_scenario()`, `all_scenarios()`) and WP03's
  alien source/manifest/reference parser. May author small parser **variants** inline in the
  test files (or tiny fixtures under its own owned test files) to force each edge case.
- Read [spec.md](../spec.md) §"Edge cases" and Success Criteria SC-002/003/004, plus
  [contracts/drawer-grading-contract.md](../contracts/drawer-grading-contract.md).
- **Ownership note:** this WP owns only test files. It may define throwaway parser variants
  *inside* those test modules; it must not edit `intake_strategy.py`, `grader.py`, or the
  WP03 fixtures (import/compose them instead).

## Subtasks

### T017 — Mis-filed intake row fails `loaded` (SC-002)

**Steps** (`tests/test_intake_scenario_drawer.py`):
1. Use a parser variant that writes a nutrition/supplement row into the **observation**
   tables (`hp.fact_*`) instead of the intake drawer.
2. Run the full intake-scenario grade; assert `loaded` is **false** (the row is absent from
   intake boundary truth) and the overall verdict fails. No cross-drawer coercion is scored
   as success (FR-006).

### T018 — Unmappable field declared, not dropped (SC-004)

**Steps** (`tests/test_intake_scenario_drawer.py`):
1. Variant A declares the `note` gap → `honest_about_gaps` true.
2. Variant B **silently drops** `note` (doesn't load it, doesn't declare it) → assert
   `honest_about_gaps` **false** (silent drop detected by manifest reconcile, FR-005).

### T019 — Renamed-but-consumed column is accounted (two distinct controls)

> **Two different controls — assert each separately (review fix).** `self_reconcile` is
> manifest-blind: it counts a column accounted iff it is in `MAPPED_SOURCE_COLUMNS` **or**
> declared as a gap. The grader's `honest_about_gaps` is a different check: manifest-derived
> truth vs (loaded ∪ declared). The renamed-but-consumed column must pass **both**, and the
> test asserts them **independently**, not as a blended "one of these passed".

**Steps** (`tests/test_intake_reconcile_renamed.py`):
1. The reference parser consumes `logged_at_us` under a different internal name (the event
   timestamp) and lists it in `MAPPED_SOURCE_COLUMNS`.
2. **Control A — `self_reconcile`:** assert `logged_at_us` is in `accounted` (because it is in
   `MAPPED_SOURCE_COLUMNS`) and **not** in `unaccounted`.
3. **Control B — grader `honest_about_gaps`:** assert the grader does **not** flag
   `logged_at_us` as a silent-drop (it is loaded/consumed per the manifest), and the rule passes.
4. Both controls are exercised end-to-end (real sandbox→grade); this is the spec edge case
   "renamed-but-consumed field".

### T020 — Intake-only vs both drawer targets

**Steps** (`tests/test_scenario_drawer_targets.py`):
1. Parser returns **intake-only** → graded on the intake drawer; observation absence is not
   a failure.
2. Parser returns **both** observation and intake → graded on each target drawer in the
   scenario's target set (the harness no longer assumes observation).
3. Prove `normalize_parse_output` separation flows correctly through grading.

### T021 — No-fork + ≥2 scenarios structural test

**Steps** (`tests/test_scenario_no_fork.py`):
1. **Structural (NFR-005):** read the source of `grade()` (and the shared grade orchestration)
   and assert it contains **no** per-drawer token (`intake`, `nutrition`, `supplement`,
   `fact_`, or an `if .*drawer` switch). Divergence must live behind the strategy.
2. **≥2 scenarios (NFR-006 / SC-003):** assert `all_scenarios()` returns ≥2 scenarios and
   that both grade through the **one** `grade()` entry with no scenario-specific code path.
