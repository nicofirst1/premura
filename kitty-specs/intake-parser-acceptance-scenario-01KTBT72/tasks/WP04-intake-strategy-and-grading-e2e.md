# WP04 — Intake drawer strategy + scenario registry + happy-path e2e

## Objective

Build the intake `DrawerGradingStrategy` (intake-table boundary truth + the WP02 runtime
checker + intake gap reconcile), expose a scenario registry listing **both** the observation
and intake scenarios, and prove the layer-1 full pass end-to-end: the reference intake parser
over the alien source scores a clean three-rule verdict. This is the first user-visible
intake value.

## Context

- Depends on WP01 (`Scenario`/`DrawerGradingStrategy`/`grade()` strategy param), WP02
  (`check_intake_runtime_contract`), WP03 (alien source + manifest + reference parser).
- Read [contracts/drawer-grading-contract.md](../contracts/drawer-grading-contract.md),
  [research.md](../research.md) D3/D4, and [data-model.md](../data-model.md).
- Intake homes: `hp.nutrition_intake_event` / `hp.nutrition_intake_item` /
  `hp.nutrition_quantity` and `hp.supplement_intake_event` / `hp.supplement_item` /
  `hp.supplement_dose` (migration 004). Boundary truth counts rows here.

## Subtasks

### T012 — Intake boundary-truth reader

**Steps** (in new `src/premura/harness/intake_strategy.py`):
1. Implement `boundary_truth(conn)` that counts persisted rows in the intake event tables
   (and derives the "present" keys the `loaded` rule needs, e.g. by supplement matcher /
   nutrition quantity key) **from the warehouse**, never from the parser's report.
2. A nutrition/supplement row that landed in `hp.fact_*` is, by construction, absent here →
   it cannot satisfy `loaded` (FR-006; the failing-case proof is WP05).

### T013 — Intake gap reconciler

**Steps**:
1. Implement `gap_set(manifest, provenance, boundary_truth)`: derive the true gap set from
   the manifest (source columns minus accounted), where accounted = loaded (warehouse truth)
   ∪ declared (`unmapped_metrics` + `skipped_rows[*].raw_field`). A column neither loaded nor
   declared is a silent-drop → present in the returned gap set (fails `honest_about_gaps`).
2. Declared metadata is **evidence to verify**, never proof (FR-005).

### T014 — Assemble the intake `DrawerGradingStrategy`

**Steps**:
1. Wire `boundary_truth` (T012) + `runtime_check` = `check_intake_runtime_contract` (WP02) +
   `gap_set` (T013) into an `IntakeStrategy` implementing the `DrawerGradingStrategy` protocol.
2. Provide `intake_scenario()` returning a `Scenario` for the alien source (source +
   manifest + reference parser + this strategy).

### T015 — Scenario registry

**Steps** (new `src/premura/harness/scenario_registry.py`):
1. `all_scenarios()` returns `[observation_scenario(), intake_scenario()]` (imports the WP01
   observation factory + the T014 intake factory). This is the ≥2-scenarios surface WP05's
   structural test asserts over. Keep it import-light.

### T016 — Layer-1 happy-path e2e

**Steps** (`tests/test_intake_scenario_grading.py`):
1. End-to-end (no mocks of the grade path): run the **reference intake parser** over the
   alien source through the real sandbox→ingest→grade flow for the intake scenario.
2. Assert a **full three-rule pass**: `loaded` true (intake rows present), `runtime_valid`
   true (intake clauses), `honest_about_gaps` true (the `note` gap is declared). (SC-001.)
3. Offline/deterministic — no network, no model server (NFR-001).
