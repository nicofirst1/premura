# CLAUDE.md

Premura is **operated and extended by AI agents**, for a human beneficiary
(agent-first in execution, human-first in purpose). Two rules govern every
spec, plan, contract, and doc:

1. **Agent-first** — the agent is the primary operational client; don't design
   human forms/dashboards as the default. Capture and analysis are agent-mediated.
2. **Design a level above — guide, don't enumerate** — write bounded abstractions
   agents fill in (registries, rubrics, contracts), not exhaustive lists of
   domains/metrics/questions/policies. Self-check: *does this hardcode a list
   where it should define the rule for adding to the list?*

Before writing or reviewing any spec or plan, read
[`docs/product/DOCTRINE.md`](docs/product/DOCTRINE.md).

The full agent router, vocabulary rules, and standards-first guidance live in
[`AGENTS.md`](AGENTS.md) — read it next.
